Duplicate Finder — Mode d'emploi (Windows)

Guide pas-à-pas : télécharger, préparer et lancer le script duplicate_finder.py afin de détecter et gérer les fichiers en double sur PC, clé USB ou smartphone Android via ADB.

Avant de commencer — ce dont vous avez besoin

  • Un ordinateur Windows (Windows 10/11 recommandé).
  • Accès administrateur non requis, mais vous devez pouvoir télécharger des fichiers.
  • Le fichier duplicate_finder.py

Étapes rapides

  1. Télécharger Python 3.8+ (si absent)
    Si Python n'est pas installé, téléchargez l'installeur Windows officiel et installez-le.
    Télécharger Python (page officielle)
    Pendant l'installation : cochez absolument "Add Python to PATH" puis Install Now.
  2. Option Android (ADB) — seulement si vous voulez scanner un téléphone
    Téléchargez Platform Tools (ADB) depuis la page officielle et extrayez le dossier dans C:\platform-tools (emplacement recommandé).
    Télécharger Platform Tools
  3. Activer l'accès USB sur le téléphone (si vous scannez Android)
    Sur le téléphone : Paramètres → À propos du téléphone → Numéro de build : tapez 7 fois pour activer les options développeur. Ensuite : Paramètres → Options développeur → USB debugging (activez). Lors de la connexion USB, choisissez le mode Transfert de fichiers (MTP) si possible.
  4. Lancer le script
    Double cliques sur le fichier duplicate_finder.py :
    Une fenêtre s'ouvrira : choisissez Local folder... ou Removable drive... ou Android via ADB...
  5. Utiliser l'explorateur Android intégré
    Si vous cliquez Android via ADB..., une petite fenêtre présente le contenu de /sdcard. Double-cliquez sur les dossiers pour descendre et cliquez sur Select this folder pour lancer le scan du dossier choisi (récursif : le script explore tous les sous-dossiers automatiquement).
    Remarque : Le script doit parcourir de milliers de fichiers pour trouver adb; veuillez patienter quelques instants...
  6. Résultats et action
    La fenêtre du script affiche le journal (log) en temps réel et la barre de progression. À la fin, le script vous proposera soit de supprimer les doublons soit de les déplacer dans un dossier nommé duplicates_review (à côté du fichier conservé).

Détails utiles (rapide & concret)

  • Pas de modules à installer : le script utilise uniquement Python standard.
  • ADB : nécessaire uniquement si vous voulez scanner un téléphone. Placer platform-tools dans C:\platform-tools accélère la détection automatique.
  • Exécution : préférez PowerShell pour voir les messages (Shift+clic droit → Ouvrir PowerShell ici).
  • Annuler : le bouton Cancel est actif pendant le scan pour arrêter proprement l'opération.
Remarque : le script fait tout le travail automatiquement — il vous demande uniquement de choisir la source (dossier / lecteur / dossier sur téléphone). Suivez simplement les fenêtres ; aucune connaissance en programmation n'est nécessaire.

Questions fréquentes

  1. Je n'ai pas adb et je veux scanner mon téléphone — que faire ?
    Téléchargez Platform Tools (lien ci-dessus) et extrayez dans C:\platform-tools. Ensuite relancez le script. Le script tentera automatiquement de détecter adb.
  2. Je vois mon téléphone dans l'explorateur Windows mais il n'est pas détecté par le script
    L'Explorateur utilise MTP (protocole spécial). Pour le script, le téléphone doit autoriser le débogage USB (USB debugging) et adb doit être détectable. Suivez l'étape d'activation USB ci-dessus.
