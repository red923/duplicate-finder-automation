Overview — Vue d'ensemble

This bilingual guide explains prerequisites and steps to run duplicate_finder.py on Windows, scan removable drives, and optionally scan Android devices using ADB. / Ce guide bilingue explique les prérequis et les étapes pour exécuter duplicate_finder.py sous Windows, analyser des lecteurs amovibles et, optionnellement, des appareils Android via ADB.

Quick notes (English)

  • System: Windows 10 / 11 (recommended)
  • Script: duplicate_finder.py
  • Python: 3.8+ recommended
  • ADB (Platform Tools) only needed to scan Android phones

Notes rapides (Français)

  • Système : Windows 10 / 11 (recommandé)
  • Script : duplicate_finder.py
  • Python : 3.8+ recommandé
  • ADB (Platform Tools) seulement nécessaire pour les téléphones Android

Quick start — Démarrage rapide

English — Fast path

  1. Install Python 3.8+ and check "Add Python to PATH" during installation.
  2. Place duplicate_finder.py in a folder (e.g. C:\tools\duplicate-finder).
  3. Open PowerShell in that folder (Shift + Right click → "Open PowerShell window here").
  4. Run: 
    python duplicate_finder.py
  5. Choose the target in the GUI: Local folder, Removable drive, or Android via ADB. Review results before deleting.

Français — Chemin rapide

  1. Installez Python 3.8+ et cochez « Add Python to PATH » pendant l'installation.
  2. Placez duplicate_finder.py dans un dossier (p. ex. C:\tools\duplicate-finder).
  3. Ouvrez PowerShell dans ce dossier (Shift + clic droit → "Open PowerShell window here").
  4. Exécutez : 
    python duplicate_finder.py
  5. Choisissez la source dans l'interface : Local folder, Removable drive ou Android via ADB. Vérifiez les résultats avant toute suppression.

Android via ADB — Setup / Android via ADB — Configuration

English — Steps

  1. Download Platform Tools and extract the platform-tools folder (recommended location: C:\platform-tools).
  2. Connect the phone via USB and enable Developer Options → USB debugging. Authorize the PC when prompted.
  3. If the script cannot detect adb, add C:\platform-tools to your PATH or run the script from a shell where adb is available.
  4. Note: Windows Explorer showing the phone via MTP does not imply ADB access — USB debugging is required.

Français — Étapes

  1. Téléchargez Platform Tools et extrayez le dossier platform-tools (emplacement recommandé : C:\platform-tools).
  2. Connectez le téléphone en USB et activez Options pour les développeurs → Débogage USB. Autorisez le PC quand il le demande.
  3. Si le script ne trouve pas adb, ajoutez C:\platform-tools au PATH ou lancez le script depuis un shell où adb est accessible.
  4. Remarque : le téléphone visible via MTP dans l'Explorateur ne signifie pas que ADB est disponible — le débogage USB est nécessaire.

Troubleshooting — Dépannage

English

  • If the script doesn't run: verify python --version in PowerShell.
  • If adb is missing: check C:\platform-tools\adb.exe and PATH.
  • Long scans: many small files or large drives take time — test on a small folder first.
  • Permissions: admin rights are usually not required. Avoid running as admin unless needed.

Français

  • Si le script ne démarre pas : vérifiez python --version dans PowerShell.
  • Si adb est introuvable : contrôlez C:\platform-tools\adb.exe et le PATH.
  • Scans longs : de nombreux petits fichiers ou de gros volumes prennent du temps — testez d'abord sur un petit dossier.
  • Permissions : les droits administrateur ne sont généralement pas nécessaires. N'exécutez en admin que si indispensable.

FAQs — Questions fréquentes

English

  • Q: I don't have ADB — can I still scan the phone?
    A: Not reliably. ADB + USB debugging is required for filesystem access via the script.
  • Q: Will the script permanently delete files?
    A: Yes — deletion is permanent unless you use recovery tools. Always backup first.
  • Q: Does the script require external Python packages?
    A: The original README indicates it uses the standard library only. Check the script help for changes.

Français

  • Q : Je n'ai pas ADB — puis-je scanner le téléphone ?
    R : Pas de manière fiable. ADB + débogage USB sont requis pour l'accès au système de fichiers via le script.
  • Q : Le script supprime-t-il définitivement les fichiers ?
    R : Oui — la suppression est permanente sauf recours aux outils de récupération. Sauvegardez toujours au préalable.
  • Q : Le script nécessite-t-il des paquets Python externes ?
    R : Le README original indique l'utilisation uniquement de la bibliothèque standard. Vérifiez l'aide du script si nécessaire.
